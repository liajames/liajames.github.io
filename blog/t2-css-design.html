CSS Concepts

I wish I knew why I was struggling...

Jan. 25, 2014

This week I've learned a lot about how to manipulate the elements on a webpage through CSS and HTML. One concept that I struggled with the most it seems is the display property.

When something is challenging for me to figure it out, I find it helpful to try to explain it so that I can really break down the concept. Want to learn the difference between display inline vs. inline blocks? Great! Let's chat about it.

Display: inline

If you tag an element with Display: Inline you're telling that element that it won't have a width or height set.It will respect it's left and right margins and padding, but not the top or bottom.

Display: Inline allows other elements to sit to the left and right of the element you're currently manipulating. This is nice for lining elements up next to one another.

Display: inline-blocks

Display: inline-blocks has a little bit more flexiblity when manipulating an element. With this function, the element is able to be positioned with elements to its left and right, respect it's top and bottom margins and padding, as well as adjust height and width.

Because of this, you're able to create a grid of elements and align your elements without creating extra work for yourself.

The trick in deciding which to use is how you wish to position your elements. It's important to carefully evaluate how these will line up and be laid out before deciding which display property to use.
